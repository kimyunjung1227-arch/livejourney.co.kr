# 📱 Android 에뮬레이터 설정 가이드

## ❌ 에뮬레이터가 실행되지 않는 경우

### 1️⃣ AVD(Android Virtual Device) 생성

**Android Studio에서:**

1. **Tools → Device Manager** 클릭
2. **Create Device** 버튼 클릭
3. **Phone** 카테고리 선택
4. **Pixel 5** 또는 **Pixel 6** 선택 → **Next**
5. **시스템 이미지 선택:**
   - **Release Name:** API 33 (Android 13) 또는 API 34 (Android 14)
   - **Download** 버튼 클릭 (이미지가 없다면)
   - 다운로드 완료 후 **Next**
6. **AVD 이름 설정** → **Finish**

### 2️⃣ 하이퍼바이저 활성화 (Windows)

**BIOS 설정:**
- 컴퓨터 재시작 → BIOS 진입 (보통 F2, F10, Del 키)
- **Virtualization Technology** 또는 **Intel VT-x** / **AMD-V** 활성화
- 저장 후 재부팅

**Windows 기능:**
- Windows 검색 → **Windows 기능 켜기/끄기**
- **Hyper-V** 체크 (Windows 11 Pro/Enterprise)
- 또는 **Windows Hypervisor Platform** 체크

### 3️⃣ HAXM 설치 (Intel CPU인 경우)

1. Android Studio → **Tools → SDK Manager**
2. **SDK Tools** 탭
3. **Intel x86 Emulator Accelerator (HAXM installer)** 체크
4. **Apply** → 설치 완료 후 재시작

### 4️⃣ 시스템 요구사항 확인

**최소 요구사항:**
- RAM: 8GB 이상 (16GB 권장)
- 디스크 공간: 10GB 이상 여유 공간
- CPU: 64-bit 프로세서

## ✅ 대안: 실제 디바이스 사용 (추천!)

에뮬레이터보다 **실제 안드로이드 폰**을 사용하는 것이 더 빠르고 안정적입니다.

### USB 디버깅 활성화

**안드로이드 폰에서:**

1. **설정 → 휴대전화 정보**
2. **빌드 번호** 7번 연속 탭 (개발자 옵션 활성화)
3. **설정 → 개발자 옵션**
4. **USB 디버깅** 활성화
5. USB 케이블로 PC 연결
6. 폰에서 **USB 디버깅 허용** 팝업 확인

**Android Studio에서:**

1. **Run** 버튼 클릭
2. 연결된 디바이스 목록에서 **실제 폰** 선택
3. 앱 실행!

## 🔧 문제 해결

### 에러: "HAXM is not installed"
→ HAXM 설치 또는 Hyper-V 비활성화

### 에러: "Emulator process was killed"
→ RAM 부족 또는 하이퍼바이저 문제

### 에러: "No system images installed"
→ SDK Manager에서 시스템 이미지 다운로드

### 에뮬레이터가 너무 느림
→ 실제 디바이스 사용 권장

## 📞 빠른 해결

**가장 빠른 방법:**
1. 실제 안드로이드 폰 연결
2. USB 디버깅 활성화
3. Android Studio에서 실행

이 방법이 에뮬레이터보다 훨씬 빠르고 안정적입니다!
