# 📤 GitHub에 올릴 파일 요약

## ✅ 반드시 올려야 할 것

### 1. 소스 코드 (가장 중요!)
```
src/
├── pages/          ← 모든 화면 파일들
├── components/     ← 컴포넌트들
├── utils/          ← 유틸리티 함수들
├── api/            ← API 호출 코드들
├── contexts/       ← Context API
├── App.jsx
├── main.jsx
└── index.css
```

### 2. 정적 파일
```
public/
├── favicon.svg
├── logo.svg
├── livejourney-logo.png
├── manifest.json
└── _redirects
```

### 3. 설정 파일 (필수!)
- ✅ `package.json` - 라이브러리 목록
- ✅ `package-lock.json` - 버전 고정
- ✅ `vite.config.js` - 빌드 설정
- ✅ `tailwind.config.js` - 스타일 설정
- ✅ `postcss.config.js` - CSS 처리 설정
- ✅ `index.html` - 메인 HTML 파일
- ✅ `.gitignore` - 제외할 파일 목록
- ✅ `capacitor.config.json` - Capacitor 설정

### 4. 문서 파일 (선택사항이지만 추천)
- ✅ `README.md` - 프로젝트 설명
- ✅ 기타 `.md` 파일들 (가이드 문서들)

---

## ❌ 절대 올리지 말아야 할 것

`.gitignore` 파일이 있어서 자동으로 제외되지만, 확인용:

- ❌ `node_modules/` - npm install로 설치됨
- ❌ `dist/` - 빌드 결과물 (npm run build로 생성)
- ❌ `.env` - 환경 변수 (민감한 정보)
- ❌ `android/`, `ios/` - 네이티브 빌드 파일
- ❌ `.vite/` - 캐시 파일
- ❌ 로그 파일들

---

## 🚀 올리는 방법 (간단!)

### 방법 1: 자동으로 올리기 (추천!)

`.gitignore` 파일이 이미 설정되어 있어서, 그냥 이렇게 하면 필요한 파일만 자동으로 올라갑니다:

```bash
cd web
git add .
git commit -m "Upload project files"
git push origin main
```

### 방법 2: 확인하면서 올리기

```bash
cd web
git status
```

이 명령어를 실행하면 올라갈 파일 목록이 나옵니다.
`node_modules`, `dist`, `.env` 같은 것들이 목록에 **없으면** 정상입니다!

## 💡 핵심 정리

**올리는 것:**
- ✅ 모든 `.jsx`, `.js`, `.css` 파일들 (`src/` 폴더)
- ✅ 이미지, 로고 파일들 (`public/` 폴더)
- ✅ 설정 파일들 (`package.json`, `vite.config.js` 등)

- ❌ `node_modules/` (너무 크고, npm install로 설치됨)
- ❌ `dist/` (빌드 결과물, npm run build로 생성됨)
- ❌ `.env` (민감한 정보)
