# 📢 카페 회원 공유 가이드

LiveJourney를 카페 회원들과 공유하는 방법입니다.

## 🚀 빠른 배포 (5분)

### 1단계: GitHub 저장소 설정

1. GitHub 저장소로 이동: `https://github.com/kimyunjung1227-arch/livejourney.com`
2. **Settings** → **Pages** 클릭
3. **Source**에서 **"GitHub Actions"** 선택
4. **Save** 클릭

### 2단계: Actions 권한 설정

1. **Settings** → **Actions** → **General** 이동
2. **"Workflow permissions"**에서 **"Read and write permissions"** 선택
3. **Save** 클릭

### 3단계: 배포 시작

코드를 푸시하면 자동으로 배포됩니다:

```bash
git push origin master
```

또는 GitHub에서 **Actions** 탭 → **"Deploy to GitHub Pages"** → **"Run workflow"** 클릭

### 4단계: 배포 확인

1. **Actions** 탭에서 배포 진행 상황 확인 (5-10분 소요)
2. 배포 완료 후 접속 URL 확인:
   ```
   https://kimyunjung1227-arch.github.io/livejourney.com/
   ```

## 📱 카페에 공유할 메시지

배포가 완료되면 카페에 다음 메시지를 공유하세요:

```
🎉 LiveJourney 웹 앱이 배포되었습니다!

📱 접속 주소: https://kimyunjung1227-arch.github.io/livejourney.com/

✨ 주요 기능:
- 📸 실시간 여행 정보 공유
- 🗺️ 지도 기반 탐색
- 🤖 AI 자동 태그 생성 (날씨 정보 포함)
- 🏆 뱃지 시스템
- 🔍 스마트 검색

💡 사용 방법:
1. 위 링크를 클릭하여 접속
2. 모바일에서 접속하면 앱처럼 사용 가능
3. 소셜 로그인 또는 테스터 계정으로 시작

📌 팁: 모바일 브라우저에서 "홈 화면에 추가"하면 앱처럼 사용할 수 있습니다!
```

## ✅ 배포 확인 체크리스트

- [ ] GitHub Pages 설정 완료 (Settings → Pages)
- [ ] Actions 권한 설정 완료 (Settings → Actions → General)
- [ ] 배포 워크플로 실행 완료 (Actions 탭에서 확인)
- [ ] 배포된 URL 접속 테스트
- [ ] 모든 페이지 정상 작동 확인
- [ ] 모바일에서 접속 테스트

## 🔄 업데이트 방법

코드를 수정한 후 다시 배포하려면:

```bash
git add .
git commit -m "업데이트 내용"
git push origin master
```

자동으로 배포가 시작됩니다!

## ❓ 문제 해결

### 404 에러가 나는 경우
- **Settings** → **Pages**에서 Source가 올바르게 설정되었는지 확인
- **Actions** 탭에서 배포가 성공했는지 확인
- 브라우저 캐시 삭제 후 다시 시도

### 배포가 안 되는 경우
- **Actions** 탭에서 에러 메시지 확인
- **Settings** → **Actions** → **General**에서 권한 확인
